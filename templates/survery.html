확장 프로그램 개발 _ 설문조사

개요

  • 단어 검색 확장 프로그램 원스탭은 프로토타입 개발 후 크롬 웹 스토어 등록 준비 중
  • 정식 버전 개발 전에 유저들이 프로덕트에 어떻게 느끼는지 알아야 하는 상황
  • 이에 프로덕트 공개 시현 후 청중들에게 설문조사 참여 요청
  • 원스탭은 유저의 기능성, 유용성 니즈 만족을 디자인, 가치관 니즈 만족보다 잘 만족시킴
  • 결론적으로 기능 중심의 정식 버전 출시에 자원을 더욱 투자해야 한다고 판단

개요

  • 시기 : 2024.11.25.14:00
  • 장소 : 동아일보 미디어 데이터 과정 부트캠프 수료장
  • 대상 : 수료생 및 관계자 38인
  • 방식 : 프로그램 설명 및 시현 후 🔗구글폼으로 데이터 수집

과정

  1. 기획 배경

    • 기사의 어려운 용어로 독자가 독해에 어려움을 겪음
    • 웹 기사는 지면 한계로 각주 제공이 불가능한 상황
    • 간편 검색으로 즉시 뜻 확인 가능한 “원스탭” 프로토타입 개발
    • 유저들의 프로덕트 예상 만족도 데이터 파악을 위한 유저 리서치 진행

  1. 설계 및 수집¹

    • 사용자 기본 정보 (성별, 연령대)
    • 기사 독해 중 단어 검색 여부 및 빈도
    • 단어 검색 시 불편 요소 파악 (탭 이동, 검색 결과 불만족 등)
    • 원스탭 기능 이해도 및 직관성 평가
    • 기존 방식 대비 원스탭의 예상 만족도 평가 (사용 편의성, 예측 가능성 등)
    • 원스탭 추천 의향 및 추천/비추천 이유
    • 기타 사용자 의견 및 개선점

  1. 분석 및 시각화

  • 정량 분석 : 객관식 및 체크박스 문항

    • 기초 통계 : 평균, 최대 최솟값, 분산, 표준편차 등
    • 고급 분석: 상관관계 분석 등
    • 툴: 구글폼 대시보드, Pandas를 활용한 데이터 전처리 및 분석
    • 주의 사항: 결측치, 중복 값 처리

  • 정성 분석 : 단문형, 장문형 문항

    • 감성 분석 : 긍정/부정/중립 분석
    • 빈도 분석 : 고빈도, 저빈도 단어 출력
    • 툴 : Kiwiepy, OpenAI API를 활용, ML모델
    • 주의: 한국어 관련 사용자 단어 구축, API 사용 비용 문제

  • 시각화 :

    • 기본 : 구글폼 대시보드 활용 ( xls csv 추출 가능)
    • 정량 : Matplotlib을 활용한 데이터 시각화 (꺾은선 그래프 등)
    • 정성: Wordcloud를 활용한 워드 클라우드
    • 예시: 성별 간 단어 검색 빈도 차이 분석 및 시각화
  1. 결과 해석

  • 응답 : 1명 ( 2.6% 응답률)

    • 인적 사항 : 20대 중반 남성
    • 핵심 평가(5번 문항): 기능, 유용성 4점 / 디자인, 가치관 3점
    • 해석 : 원스탭은 기능 관련 니즈 만족을 상대적으로 잘 시킨다.
    • 결론 : 원스탭의 세일즈 포인트는 감성 요소보다 실질 요소가 되어야 한다.

  • 후속 리서치 작업

    • 구글 폼 오픈을 통한 추가 데이터 수집
    • 컨택을 통한 심층 유저 인터뷰 가능( In-depth Interview)
    • 백엔드개발을 통한 검색 빈도수 등에 대한 개인 로그 분석 가능
    • Scikit-Learn등 ML 모델을 활용한 인적 사항별 만족도 예측 가능
