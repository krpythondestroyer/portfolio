확장 프로그램 개발

개요
  • 기사의 어려운 용어로 독자가 독해에 어려움을 겪음
  • 웹 기사는 지면 한계로 각주 제공이 불가능한 상황
  • 간편 검색으로 뜻을 즉시 확인 가능한 프로그램 개발
  • 간단한 설명 자료 작성 및 설문조사 수집 중
  • 독자들의 콘텐츠 독해력 상승 및 편의성 증대 기대
업무
  • 기간: 2024.10.07 ~ 2024.11.25 (8주)
  • 방식: MVP 모델(레퍼 →구상 → 핵심 → 부가)
  • 인원: 동아일보 미디어 프론티어 과정 1인
  • 일정: Notion
  • 기여: 파이널 개인 프로젝트
개발
  • 핵심 : 긁고 - 검색 기능
    • 우리말샘,구글,OpenAI → 네이버
    • CORS 문제 해결을 위한 퍼미션 설정
    • 크롬 로컬 스토리지 활용
    • IFRAME → SIDEPANEL
  • 부가 : 수정 및 링크 한 탭 이동 기능
    • 강조 : strong 태그
    • 삭제 : 크롬 로컬 스토리지값 삭제
    • 원문 이동 :API에서 파싱된 URL 이동
    • 북마크 이동 : 검색시 저장된 URL 이동
기타
  • "디지털 트윈" 검색 예시
    • 화면.jpg, 내용.jpg
  • 설문조사 :
      워드 초안 → 구글폼🔗
  • 측정 : 기간별 총 단어 검색 수
    • 제공 : "네이버 API 통계"
  • 설명 자료.jpg
    • 구조 : 정의, 배경, 장점, 타겟, 유도
  • 플로우 도식화
    • 유저.jpg, 데이터.jpg
각주
*API 연동 및 정보 출력 - 각 API 소개
  • 우리말샘 : 국립국어원에서 제공하는 단어 사전 오픈 API다. 무료라는 장점을 가지지만 단순 사전 검색으로 정보 제공 유연성이 너무 떨어져 보류되었다. 🔗
  • 구글 Custom Search Json: 구글에서 제공하는 스니펫형 API로 검색 유연성은 높지만 부분 유료이다.(하루 100쿼리까지 무료, 이후 1000쿼리당 5달러, 총 10000쿼리 제한). 스니펫이라는 원사이트 파싱형 정보 제공 방식이 단어 검색 정보 제공이라는 본 프로젝트 기획의도에 부적합해 보류되었다. 🔗
  • OpenAI gpt-4o-mini: OpenAI사에서 제공하는 LLM형 API로 프롬프트, 파인 튜닝 설정 측면에서 정보 출력 유연성은 아주 높지만 부분 유료이다. (입력, 출력 토큰 100만개당 각각 0.15$, 0.60$, 웹 크롤링 및 감성 분석 프로젝트 진행시 사용 금액 500원 미만) LLM이라는 생성형 정보 제공 방식이 기획이 단어 검색 정보 제공이라는 본 프로젝트 기획의도에 부적합해 보류되었다. 🔗
  • 네이버 Search API: 네이버에서 제공하는 백과사전 API로 무료이지만 하루 25000회 검색 제한이 있다. 백과사전식 정보 제공이라는 측면에서 검색 결과 유연성이 높고 기획 의도에도 적합해 채택되었다. 🔗
