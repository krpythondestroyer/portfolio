확장 프로그램 개발

🌐
개요
  • 기사의 어려운 용어로 독자가 독해에 어려움을 겪음
  • 웹 기사는 지면 한계로 각주 제공이 불가능한 상황
  • *간편 검색으로 즉시 뜻 확인 가능한 프로그램 개발
  • 독자들의 콘텐츠 독해력 상승 및 편의성 증대 기대
  • 간단한 설명 자료 작성 및 설문조사 수집 중
업무
  • 기간: 2024.10.07 ~ 2024.11.25 (8주)
  • 방식: MVP
  • 인원: 동아일보 미디어 데이터 과정 1인
  • 일정: Notion
  • 기여: 파이널 개인 프로젝트
단계
  1. 기획
    • 문제 발견 : 독자 어휘력, 문해력 문제 등
    • 아이디에이션 : 사이트 개설 → 익스텐션 개발
    • 구상 : 링크 형태 → 각주 형태
    • 레퍼런스 : 팝업 방식 → 화면 분할 방식
  1. 핵심 개발 : 긁고 - 검색 기능
    • **API : 우리말샘,구글,OpenAI → 네이버
    • 화면 : IFRAME → SIDEPANEL
    • 데이터 : 윈도우 로컬 → 크롬 로컬 스토리지
    • 통신 : 백엔드 개발 → 퍼미션 설정
  1. 부가 개발: 수정 및 링크 한 탭 이동 기능
    • 강조 : strong 태그
    • 삭제 : 크롬 로컬 스토리지값 삭제
    • 원문 이동 :API에서 파싱된 URL 이동
    • 북마크 이동 : 검색시 저장된 URL 이동
  1. 개발 후
    • 설문조사 : 🔗구글폼
      • 상세보기
    • 측정 : 기간별 총 단어 검색 수
      • 제공 : "네이버 API 통계"
    • 설명 자료.jpg
      • 구조 : 정의, 배경, 장점, 타겟, 유도
각주
*
  • "디지털 트윈" 검색 예시 :화면.jpg, 내용.jpg
  • 플로우 도식화 : 유저.jpg, 데이터.jpg
**
  • 🔗우리말샘 : : 국립국어원에서 제공하는 단어 사전 오픈 API로, 무료라는 장점이 있지만 검색 결과가 지나치게 단순해 정보 제공의 유연성이 부족하다고 판단되어 보류되었다.
  • 🔗구글 Custom Search Json:구글에서 제공하는 스니펫형 API로 검색 유연성은 뛰어나지만, 부분 유료 서비스다. (하루 100쿼리까지 무료, 이후 1000쿼리당 5달러, 최대 10000쿼리 제한) 단어 검색 프로젝트의 기획 의도와 맞지 않는 스니펫 방식이라 보류되었다.
  • 🔗OpenAI gpt-4o-mini: OpenAI에서 제공하는 LLM 기반 API로, 프롬프트 및 파인 튜닝을 통해 출력 유연성은 아주 높지만 부분 유료이다. (입력, 출력 토큰 100만개당 각각 0.15$, 0.60$, 웹 크롤링 및 감성 분석 프로젝트 진행시 사용 금액 500원 미만). 생성형 AI라는 특성이 단어 검색이라는 프로그램 기획 의도에 부적합해 보류되었다.
  • 🔗네이버 Search API: 네이버에서 제공하는 백과사전 API로, 하루 25,000회 검색 제한이 있지만 무료로 이용할 수 있다. 백과사전식 정보 제공 방식이 검색 결과의 유연성을 높여주며, 프로젝트의 기획 의도와도 부합해 최종적으로 채택되었다.
